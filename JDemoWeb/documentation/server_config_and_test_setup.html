The projects JDemoWeb and JDemoWebLegacy have been tested with Tomcat 9.0.43 in the following context:
Eclipse IDE for Enterprise Java and Web Developers (includes Incubating components)
Version: 2021-03 (4.19.0)
Build id: 20210312-0638
OS: Windows 10, v.10.0, x86_64 / win32
Java version: 15.0.2

There is a bug in Spring 2.5.0 that causes an error on server starting asking for a missing config directory. The workaround that I used was detailed here and involves editing the Tomcat server launch configuration to add the environment variable SPRING_CONFIG_LOCATION with value optional:classpath:/,optional:classpath:/config/,optional:file:./,optional:file:./config/.

The JDemoWeb project includes an example of including another Eclipse project as a referenced library. In Rational Application Developer this is more automated (called a "Web Library Project"), but for the base Eclipse configuration you'll need to add the project JDemoWebLib in the "User Entries" classpath in the server launch configuration.

To verify configuration for JDemoWeb, hit the URL http://localhost:8080/sbw/junit which uses the JUnit text running to run some JUnit tests (a bit unorthodox, but this is just a demo). To verify configuration for JDemoWebLegacy, use http://localhost:8080/springweb/cust. Note that some of these tests are expecting test data in a MySQL database which you'll need to create and populate with a bit of test data (TODO: insert data in test setup, better documentation on database creation).
